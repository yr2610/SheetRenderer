# テンプレートを使う

このチュートリアルでは、同じ構造を何度も書く資料を短く保つ方法を説明します。

まだ基本的なシートを書いたことがない場合は、先に
[はじめての SheetRenderer パーサー](parser-tutorial.md)を読んでください。

## 1. 同じ構造はテンプレートにする

同じ確認項目を何度も書く場面を考えます。

```text
# 確認
- 1回目
  - 申込内容を確認
  - 結果を記録
- 2回目
  - 申込内容を確認
  - 結果を記録
- 3回目
  - 申込内容を確認
  - 結果を記録
```

この程度ならコピーでも書けますが、子項目を追加するたびに 3 箇所を直すことに
なります。数が増えると、直し忘れも起きやすくなります。

そこで、繰り返す構造へ名前を付けます。

```text
# 確認
- &check()
  - 申込内容を確認
  - 結果を記録

- 1回目
- *check()
- 2回目
- *check()
- 3回目
- *check()
```

`&check()` がテンプレートの宣言です。その子項目がコピー元になります。

`*check()` がテンプレートの呼び出しです。呼び出した場所へ、コピー元の項目を
まとめて差し込みます。宣言行と呼び出し行そのものは、出力には残りません。

テンプレートを使うと、コピーしたいまとまりを 1 箇所で管理できます。

## 2. 少しだけ違う部分は引数で渡す

先ほどの例では、共通部分をテンプレートにできましたが、「何回目か」はまだ
手書きで残っています。コピー先ごとに少しだけ文言を変えたい場合は、
テンプレートへ引数を渡します。

```text
# 確認
- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check({ number: 1 })
- *check({ number: 2 })
- *check({ number: 3 })
```

呼び出し側の `{ number: 1 }` が、テンプレートへ渡すデータです。
テンプレート内の `{{number}}` は、その値に置き換わります。

名前付きの値が一つだけの場合でも、まずはこの書き方を覚えておけば十分です。

## 3. 呼び出し行が増えたら配列でまとめる

引数を使っても、回数が多いと呼び出し行が並びます。

```text
- *check({ number: 1 })
- *check({ number: 2 })
- *check({ number: 3 })
```

同じテンプレートへ渡すデータは、配列としてまとめられます。

```text
# 確認
- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check([{ number: 1 }, { number: 2 }, { number: 3 }])
```

配列の要素数だけテンプレートが展開されます。100 件あっても、テンプレートの
呼び出し行は 1 行です。

## 4. データが増えたら YAML で書く

配列が長くなると、テキストファイルの中にデータまで書くのがつらくなります。
その場合は、シート内に YAML ブロックを書きます。

````text
# 確認

```yaml
checks:
  - number: 1
  - number: 2
  - number: 3
```

- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check(checks)
````

`checks` は YAML ブロックに書いた配列です。呼び出し側では、その名前を
渡すだけで済みます。

Excel など別の資料から作ったデータも、YAML に変換しておけば同じように
テンプレートへ渡せます。構造の修正とデータの修正を分けて考えられるようになります。

## 5. 一部だけ構成を変える

データによって、一部の項目が必要だったり不要だったりすることがあります。
そのためにテンプレートを丸ごと複製する必要はありません。

````text
# 確認

```yaml
checks:
  - name: 通常申請
    extra: 添付書類も確認
  - name: 簡易申請
    extra: null
```

- &check()
  - {{name}}
    - 基本項目を確認
    - {{extra}}
      - 不足があれば担当者へ連絡

- *check(checks)
````

`通常申請` には `extra` があるので、次の構造になります。

```text
- 通常申請
  - 基本項目を確認
  - 添付書類も確認
    - 不足があれば担当者へ連絡
```

`簡易申請` の `extra` は `null` です。`{{extra}}` の項目と、その子項目は
まとめて削除されます。

```text
- 簡易申請
  - 基本項目を確認
```

項目を意図的に消したい場合は、YAML 側で `false` または `null` を明示して
ください。値を書き忘れた場合にも項目は削除されますが、その時は記述ミスや
データ不足に気づけるように警告が出ます。

ここまでで、テンプレートを使う時の基本形は完成です。

## 6. 複数シートで使うデータだけ `vars.yml` に置く

通常の YAML は、そのデータを使うシート内に書きます。データと利用箇所が近いので、
後から見返した時にも追いやすくなります。

ただし、同じ YAML を複数のシートから使いたい場合は、全シートへ同じ内容を
コピーするのも面倒です。その場合に限り、`vars.yml` へ移せます。

```text
source/
  index.txt
  vars.yml
```

`vars.yml`:

```yaml
documentTypes:
  - 申込書
  - 本人確認書類
```

`vars.yml` の値は、すべてのシートから参照できます。

```text
# 受付時確認
- &document()
  - {{}}
- *document(documentTypes)

# 最終確認
- &document()
  - {{}}
- *document(documentTypes)
```

便利ですが、何でも `vars.yml` に集めると、どのシートが何を使っているのか
分かりにくくなります。

まずはシート内の YAML を使い、複数シートで本当に共有したいものだけを
`vars.yml` に置いてください。

## 7. 一度しか使わないテンプレートを短く書く

テンプレートを一度だけ呼び出す場合、名前を付けるとかえって長くなることがあります。

```text
# 試行
- &trial()
  - {{}}回目
    - 当選しないこと

- *trial([1, 2, 3])
```

この場合は、匿名テンプレートを使うと短く書けます。

```text
# 試行
- *([1, 2, 3])
  - {{}}回目
    - 当選しないこと
```

`*(...)` の子項目が、その場限りのテンプレートになります。

配列の中身が単純な値の場合、空のプレースホルダー `{{}}` で現在の値を
参照できます。

同じテンプレートを別の場所でも呼び出したい場合は、これまでどおり名前付き
テンプレートを使ってください。
