# include ファイルの下書き作成

`下書き作成` は、エントリーファイルに書いた `include` から未作成の
テキストファイルを作るための Ribbon 機能です。パーサー構文ではなく、
パースや Excel 更新とは別に実行する補助機能です。

## 基本

エントリーファイルに、シート名にしたいコメントと `include` を書きます。

```text
// 申請内容
<<[application.txt]

// 最終確認
<<[final-check.txt]
```

`下書き作成` を実行すると、存在しない include ファイルだけを作成します。
既存ファイルは上書きしません。

```text
source/application.txt
source/final-check.txt
```

作成前に確認ダイアログが表示されます。シート名とファイル名を確認し、作成したい
下書きだけにチェックを付けて `OK` を押してください。`キャンセル` を押すと何も
作成しません。

作成する内容は、include 先のフォルダから上位フォルダへ向かって最初に見つかった
`_template.txt` を元にします。`conf.yml` の `$rootDirectory` で決まるルートフォルダ
まで探しても `_template.txt` が
見つからない場合はエラーになり、ファイルは作成されません。

## コメントの扱い

`<<[...]` 行から上へ遡り、最初に見つかった `// コメント` を `[[title]]` に渡します。
空行はないものとして扱います。

```text
// メモ
// 申請内容

<<[application.txt]
```

この場合は、一番下の `// 申請内容` が使われます。

コメントの直後に `<<[...]` が連続している場合は、一番上の include だけが作成対象です。

```text
// 申請内容
<<[application.txt]
<<[details.txt]
```

この場合、`application.txt` だけが作成対象になります。`details.txt` は、上へ遡る途中で
別の include に当たるため作成されません。

## テンプレート

`_template.txt` では、次の下書き作成用プレースホルダーを使用できます。

```text
# [[title]]

* [[name]]
  * [[path]]
```

- `[[title]]`: 直前コメントの文字列
- `[[name]]`: 作成するファイル名の拡張子なし
- `[[path]]`: include に解決されたパス

`[[title]]` は Excel のシート名として使える文字列である必要があります。空文字、
31文字を超える名前、`: \ / ? * [ ]` を含む名前などはエラーになります。
同じ下書き作成対象の中に同じシート名がある場合もエラーになります。ブックを
開いて実行した場合は、開いているブックの既存シート名との重複もエラーになります。

エラーが1件でもある場合、ファイルは1つも作成されません。
