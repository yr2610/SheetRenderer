# SheetRenderer パーサーガイド

SheetRenderer のテキストパーサーでは、見出しと箇条書きで Excel シートの構造を
記述できます。テンプレート、変数、プレースホルダーを使うと、繰り返しの多い
項目も短く書けます。

初めて使う場合は、先に[パーサーチュートリアル](parser-tutorial.md)を読むと、
各機能を使う理由と導入順序が分かりやすくなります。

## 基本構造

`#` でシート名、`*` で項目を記述します。子項目はスペース 2 個以上で
インデントします。

```text
# チェックリスト
* 申込内容
  * 氏名
  * 住所
* 添付書類
```

箇条書きの記号には `*` のほか、`-` と `+` も使用できます。

パーサーは必要に応じて `[#...]` 形式の ID をソースへ書き戻します。ID は
項目と出力結果の対応を安定させるために使われるので、通常はそのまま残してください。

## プレースホルダー

`{{name}}` で変数を参照できます。ドット区切りのパスや簡単な式も使用できます。

```text
* 商品名: {{product.name}}
* 税込金額: {{price * taxRate}}
```

テンプレート内の空のプレースホルダー `{{}}` は、スカラー値を簡潔に参照する
ための省略記法です。

```text
* &item()
  * {{}}回目
* *item([1, 2, 3])
```

この例では `1回目`、`2回目`、`3回目` が展開されます。

### 項目の省略

通常のプレースホルダーは、値が `false` または `null` の場合に項目全体を
削除します。子項目もまとめて削除されます。

```text
* 備考: {{memo}}
```

項目を意図的に消したい場合は、YAML 側で `false` または `null` を明示して
ください。省略のためにキー自体を書かない、という使い方は避けてください。

未定義の値を参照した場合も項目は削除されますが、警告を出力します。これは、
変数名のタイプミスやデータ不足によって項目が意図せず消えたことに気づくための
動作です。

### 応用: 未定義の警告を抑制する

`{{? expr}}` は、`false`、`null`、`undefined` の場合に項目全体を削除し、
未定義でも警告を出しません。

```text
* 備考: {{?memo}}
```

この記法は常用しないでください。未定義を意図的な省略として扱いたい場合には
便利ですが、記述ミスやデータ不足も見えにくくなります。通常は YAML 側で
`false` または `null` を明示し、`{{? expr}}` は未定義を許容する理由が
明確な場合だけ使用してください。

## 変数

### 固定値

`&name: value` で、その階層以下から参照できる固定値を定義できます。定義行は
出力に残りません。

```text
* &price: 1000
* &taxRate: 1.1
* 税込金額: {{price * taxRate}}
```

固定値には文字列、数値、真偽値、`null` を使用できます。計算や条件分岐が
必要な場合は `@init:` を使用してください。

### 値のリスト

`&name:` の子項目として値を並べると、テンプレートへ渡せるリストを定義できます。

```text
* &items:
  * foo
  * bar
* &item()
  * 値: {{}}
* *item(items)
```

### シート内の YAML

まとまったデータは、使うシート内の YAML ブロックへ記述できます。

````text
# チェックリスト

```yaml
items:
  - name: foo
  - name: bar
```

* &item()
  * {{name}}
* *item(items)
````

YAML ブロックの値は、そのシート内から参照できます。

### `vars.yml`

複数のシートで共通して使う値は、`vars.yml` に定義できます。`vars.yml` の値は
全シートから参照できます。

グローバルな値を増やしすぎるとデータの利用箇所が追いにくくなるため、通常は
シート内の YAML を使い、複数シートで共有する必要があるものだけを
`vars.yml` に置いてください。

## 名前付きテンプレート

`&name()` でテンプレートを宣言し、`*name(...)` で呼び出します。

```text
* &item()
  * {{name}}
    * 詳細
* *item({ name: "foo" })
```

配列を渡すと、要素ごとにテンプレートを展開します。

```text
* &item()
  * {{name}}
* *item([{ name: "foo" }, { name: "bar" }])
```

同じテンプレートを複数箇所で使う場合は、名前付きテンプレートが向いています。

## 匿名テンプレート

一度しか使わないテンプレートは、宣言と呼び出しを `*(...)` で同時に記述できます。

```text
* *([1, 2, 3])
  * {{}}回目
    * foo
```

これは、次の名前付きテンプレートと同様の展開を短く記述するための構文です。

```text
* &item()
  * {{}}回目
    * foo
* *item([1, 2, 3])
```

引数には、通常のテンプレート呼び出しと同じく、シート内の YAML や同じ階層以上で
定義した変数も渡せます。`vars.yml` に置いた共通の値も使用できます。

```text
* *(items)
  * {{name}}
```

匿名テンプレートの呼び出し行は制御用の行なので、永続 ID は書き戻されません。
展開後の項目には通常どおり ID が付きます。

## 反復

### 配列

配列を渡すと、要素ごとに 1 回ずつ展開します。

```text
* *item(["foo", "bar"])
```

スカラー値は `{{}}` または `$value` で参照できます。

### 回数

整数を渡すと、その回数だけ展開します。

```text
* *item(10)
```

同じ引数で複数回展開したい場合は `$times` も使用できます。

```text
* *item({ $times: 3, label: "foo" })
```

### range 記法

テンプレート呼び出しの引数全体を `start..end` と書くと、両端を含む整数配列
として展開します。名前付きテンプレートと匿名テンプレートの両方で使用できます。

```text
* *(1..3)
  * {{}}回目
```

これは `*([1, 2, 3])` と同じ意味です。

同じ値と降順にも対応しています。

```text
* *item(5..5)  // [5]
* *item(3..1)  // [3, 2, 1]
```

range 記法は 0 以上の整数専用です。マイナス値と小数は使用できません。
生成される要素数には `MAX_REPEAT` の上限が適用されます。デフォルトは 500 件です。

range 記法をオブジェクトや式の一部へ埋め込むことはできません。

```text
* *item({ values: 1..3 })  // 使用不可
```

### ループ情報

反復中は次の値を参照できます。

| 変数 | 内容 |
| --- | --- |
| `$index` | 0 から始まるインデックス |
| `$index1` | 1 から始まるインデックス |
| `$count` | 展開回数 |
| `$isFirst` | 最初の要素なら `true` |
| `$isLast` | 最後の要素なら `true` |
| `$isOdd` | 0 始まりのインデックスが奇数なら `true` |
| `$isEven` | 0 始まりのインデックスが偶数なら `true` |

```text
* *(1..3)
  * {{$index1}} / {{$count}}: {{}}
```

## `@init:`

`@init:` は、その場で値を初期化するための記法です。実行後、`@init:` の行は
出力に残りません。

```text
* @init: $defaults({ title: "無題", count: 0 })
* タイトル: {{title}}
* 件数: {{count}}
```

複数行にする場合は、続く行をインデントします。

```text
* @init:
    $defaults({ subtotal: 1000, taxRate: 1.1 })
    $set("total", $get("subtotal") * $get("taxRate"))
* 合計: {{total}}
```

`@init:` 内では次のヘルパーを使用できます。

| ヘルパー | 内容 |
| --- | --- |
| `$get(key)` | 現在参照できる値を取得する |
| `$set(key, value)` | 値を設定または上書きする |
| `$defaults(object)` | 未定義の値だけをまとめて設定する |

単純な固定値には `&name: value`、計算や条件分岐には `@init:` を使うと
読みやすくなります。

## 複数行テキスト

項目の続きとしてインデントした行は、同じ項目の複数行テキストになります。

```text
* 1 行目
  2 行目
  3 行目
```

従来の行末 ` +` による継続も使用できますが、新規記述ではインデントによる
継続を推奨します。

## コメント

行頭の `//`、または空白に続く `//` 以降はコメントとして扱われます。

```text
// この行はコメント
* 項目  // この部分はコメント
* URL: https://example.com/api/items
```

URL 内の `//` のように、直前に空白がないものは保持されます。

### 項目のコメントと画像

項目の末尾に `[^...]` を書くと、項目へコメントを付けられます。

```text
* 本人確認書類 [^運転免許証またはマイナンバーカード]
```

コメントを `!` で囲むと、画像ファイルを指定できます。

```text
* 記入例 [^!images/sample.png!]
```

相対パスは、その項目を書いたテキストファイルの場所を基準に解決されます。

## 使い分け

| やりたいこと | 記法 |
| --- | --- |
| 固定値を置く | `&name: value` |
| 計算した値を置く | `@init:` |
| 再利用するテンプレートを作る | `&name()` と `*name(...)` |
| 一度だけ使うテンプレートを短く書く | `*(...)` |
| 配列の各要素で展開する | `*name([...])` または `*(...)` |
| 連番で展開する | `*name(1..10)` または `*(1..10)` |
