# はじめての SheetRenderer パーサー

このチュートリアルでは、最初からすべての機能を覚える必要はありません。
まずは普通の箇条書きとして書き始め、資料が大きくなった時だけ、必要な機能を
一つずつ足していきます。

説明では、申請内容を確認するためのチェックシートを少しずつ育てます。

## 1. まずは箇条書きで書く

最初に必要なのは、シート名を表す `#` と、項目を表す `-` だけです。

```text
# 申請内容
- 申請者
  - 氏名
  - 住所
- 申請書類
  - 申込書
  - 本人確認書類

# 最終確認
- 記入漏れがないこと
- 添付漏れがないこと
```

`#` を書くと Excel のシートができます。`-` の項目は、インデントすると
親子関係になります。インデントには半角スペースを 2 個以上使ってください。

まずはこれだけで十分です。項目数が少ないうちは、一つのテキストファイルへ
全シートを書いて構いません。

### コメントを残す

ソースにだけメモを残したい場合は `//` を使います。

```text
// 法務確認後に文言を見直す
- 本人確認書類  // 必要書類は別紙を参照
```

項目そのものへ補足を付けたい場合は `[^...]` を使います。

```text
- 本人確認書類 [^運転免許証またはマイナンバーカード]
```

画像を付けたい場合は、コメント内のパスを `!` で囲みます。

```text
- 記入例 [^!images/sample.png!]
```

相対パスは、その項目を書いたテキストファイルの場所を基準にします。

## 2. 大きくなったらファイルを分ける

シートが増えると、一つのテキストファイルを探し回る時間が増えてきます。
その時は `include` を使って、内容を複数のファイルへ分けます。

```text
index.txt
source/
  application.txt
  final-check.txt
```

`index.txt` には、読み込むファイルだけを書きます。

```text
<<[application.txt]
<<[final-check.txt]
```

`source/application.txt`:

```text
# 申請内容
- 申請者
  - 氏名
  - 住所
- 申請書類
  - 申込書
  - 本人確認書類
```

`source/final-check.txt`:

```text
# 最終確認
- 記入漏れがないこと
- 添付漏れがないこと
```

パースすると、`index.txt` の場所へ各ファイルの内容を順番に差し込んだものとして
扱われます。分けても、出来上がるシートは変わりません。

ファイルがさらに大きくなった場合は、読み込まれたファイルから別のファイルを
`include` することもできます。
