# はじめての SheetRenderer パーサー

このチュートリアルでは、最初からすべての機能を覚える必要はありません。
まずは普通の箇条書きとして書き始め、資料が大きくなった時だけ、必要な機能を
一つずつ足していきます。

説明では、申請内容を確認するためのチェックシートを少しずつ育てます。

## 1. まずは箇条書きで書く

最初に必要なのは、シート名を表す `#` と、項目を表す `-` だけです。

```text
# 申請内容
- 申請者
  - 氏名
  - 住所
- 申請書類
  - 申込書
  - 本人確認書類

# 最終確認
- 記入漏れがないこと
- 添付漏れがないこと
```

`#` を書くと Excel のシートができます。`-` の項目は、インデントすると
親子関係になります。インデントには半角スペースを 2 個以上使ってください。

まずはこれだけで十分です。項目数が少ないうちは、一つのテキストファイルへ
全シートを書いて構いません。

### コメントを残す

ソースにだけメモを残したい場合は `//` を使います。

```text
// 法務確認後に文言を見直す
- 本人確認書類  // 必要書類は別紙を参照
```

項目そのものへ補足を付けたい場合は `[^...]` を使います。

```text
- 本人確認書類 [^運転免許証またはマイナンバーカード]
```

画像を付けたい場合は、コメント内のパスを `!` で囲みます。

```text
- 記入例 [^!images/sample.png!]
```

相対パスは、その項目を書いたテキストファイルの場所を基準にします。

## 2. 大きくなったらファイルを分ける

シートが増えると、一つのテキストファイルを探し回る時間が増えてきます。
その時は `include` を使って、内容を複数のファイルへ分けます。

```text
source/
  index.txt
  application.txt
  final-check.txt
```

`index.txt` には、読み込むファイルだけを書きます。

```text
<<[./application.txt]
<<[./final-check.txt]
```

`application.txt`:

```text
# 申請内容
- 申請者
  - 氏名
  - 住所
- 申請書類
  - 申込書
  - 本人確認書類
```

`final-check.txt`:

```text
# 最終確認
- 記入漏れがないこと
- 添付漏れがないこと
```

パースすると、`index.txt` の場所へ各ファイルの内容を順番に差し込んだものとして
扱われます。分けても、出来上がるシートは変わりません。

ファイルがさらに大きくなった場合は、読み込まれたファイルから別のファイルを
`include` することもできます。

## 3. 同じ構造はテンプレートにする

同じ確認項目を何度も書く場面を考えます。

```text
# 確認
- 1回目
  - 申込内容を確認
  - 結果を記録
- 2回目
  - 申込内容を確認
  - 結果を記録
- 3回目
  - 申込内容を確認
  - 結果を記録
```

この程度ならコピーでも書けますが、子項目を追加するたびに 3 箇所を直すことに
なります。数が増えると、直し忘れも起きやすくなります。

そこで、繰り返す構造へ名前を付けます。

```text
# 確認
- &check()
  - 申込内容を確認
  - 結果を記録

- *check()
- *check()
- *check()
```

`&check()` がテンプレートの宣言です。その子項目がコピー元になります。

`*check()` がテンプレートの呼び出しです。呼び出した場所へ、コピー元の項目を
まとめて差し込みます。宣言行と呼び出し行そのものは、出力には残りません。

テンプレートを使うと、コピーしたいまとまりを 1 箇所で管理できます。

## 4. 少しだけ違う部分は引数で渡す

先ほどの例では、コピーした項目に「何回目か」がありませんでした。コピー先ごとに
少しだけ文言を変えたい場合は、テンプレートへ引数を渡します。

```text
# 確認
- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check({ number: 1 })
- *check({ number: 2 })
- *check({ number: 3 })
```

呼び出し側の `{ number: 1 }` が、テンプレートへ渡すデータです。
テンプレート内の `{{number}}` は、その値に置き換わります。

名前付きの値が一つだけの場合でも、まずはこの書き方を覚えておけば十分です。

## 5. 呼び出し行が増えたら配列でまとめる

引数を使っても、回数が多いと呼び出し行が並びます。

```text
- *check({ number: 1 })
- *check({ number: 2 })
- *check({ number: 3 })
```

同じテンプレートへ渡すデータは、配列としてまとめられます。

```text
# 確認
- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check([{ number: 1 }, { number: 2 }, { number: 3 }])
```

配列の要素数だけテンプレートが展開されます。100 件あっても、テンプレートの
呼び出し行は 1 行です。

## 6. データが増えたら YAML に分ける

配列が長くなると、テキストファイルの中にデータまで書くのがつらくなります。
構造はテキスト、データは YAML に分けられます。

`source/index.txt` と同じ場所へ `vars.yml` を作ります。

```text
source/
  index.txt
  vars.yml
```

`vars.yml`:

```yaml
checks:
  - number: 1
  - number: 2
  - number: 3
```

`index.txt`:

```text
# 確認
- &check()
  - {{number}}回目
    - 申込内容を確認
    - 結果を記録

- *check(checks)
```

`checks` は `vars.yml` に書いた配列です。呼び出し側では、その名前を渡すだけで
済みます。

Excel など別の資料から作ったデータも、YAML に変換しておけば同じように
テンプレートへ渡せます。構造の修正とデータの修正を分けて考えられるようになります。

## 7. 一部だけ構成を変える

データによって、一部の項目が必要だったり不要だったりすることがあります。
そのためにテンプレートを丸ごと複製する必要はありません。

`vars.yml`:

```yaml
checks:
  - name: 通常申請
    extra: 添付書類も確認
  - name: 簡易申請
```

`index.txt`:

```text
# 確認
- &check()
  - {{name}}
    - 基本項目を確認
    - {{?extra}}
      - 不足があれば担当者へ連絡

- *check(checks)
```

`通常申請` には `extra` があるので、次の構造になります。

```text
- 通常申請
  - 基本項目を確認
  - 添付書類も確認
    - 不足があれば担当者へ連絡
```

`簡易申請` には `extra` がありません。`{{?extra}}` の項目と、その子項目は
まとめて削除されます。

```text
- 簡易申請
  - 基本項目を確認
```

先頭の `?` は、「値がない場合があっても問題ない」という印です。意図的に
省略する項目では `{{?extra}}` を使ってください。

通常の `{{name}}` に必要な値がない場合も、その項目以下は削除されます。ただし、
こちらは記述ミスやデータ不足の可能性があるため警告が出ます。

ここまでで、よく使う基本形は完成です。

## 8. 一度しか使わないテンプレートを短く書く

テンプレートを一度だけ呼び出す場合、名前を付けるとかえって長くなることがあります。

```text
# 試行
- &trial()
  - {{}}回目
    - 当選しないこと

- *trial([1, 2, 3])
```

この場合は、匿名テンプレートを使うと短く書けます。

```text
# 試行
- *([1, 2, 3])
  - {{}}回目
    - 当選しないこと
```

`*(...)` の子項目が、その場限りのテンプレートになります。

配列の中身が単純な値の場合、空のプレースホルダー `{{}}` で現在の値を
参照できます。

同じテンプレートを別の場所でも呼び出したい場合は、これまでどおり名前付き
テンプレートを使ってください。

## 9. 連番は range 記法で書く

連続した数値を並べる場合は、配列をすべて書かなくても構いません。

```text
# 試行
- *(1..10)
  - {{}}回目
    - 当選しないこと
```

`1..10` は `[1, 2, 3, ..., 10]` と同じ意味です。両端の数値を含みます。

同じ数値と降順も使えます。

```text
- *(5..5)  // [5]
- *(3..1)  // [3, 2, 1]
```

range 記法は、テンプレート呼び出しの引数として使う 0 以上の整数専用です。

## 10. 次に調べるもの

ここまでの機能だけで、通常の資料はかなり短く保てます。必要になった時は、
[パーサーガイド](parser-guide.md)で細かい仕様を確認してください。

| やりたいこと | ガイドで調べる項目 |
| --- | --- |
| 反復中に現在の回数を表示したい | ループ情報の `$index1` |
| 回数だけを指定して同じ構造を繰り返したい | 反復の整数指定 |
| 短い固定値をその場で定義したい | `&name: value` |
| 複数行の文章を書きたい | 複数行テキスト |

まずは、箇条書き、`include`、テンプレート、配列、YAML、`{{?name}}` の順で
使えるようになれば十分です。
